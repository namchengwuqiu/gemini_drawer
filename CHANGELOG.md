# Gemini 绘图插件 更新日志

## v1.9.13 (2026-06-11)

### 修复与优化
- **SSE 流式解析修复**: 修复流式响应中全局替换 `data:` 导致 Base64 图片/视频前缀被破坏的问题，并改为流结束后统一从累积内容提取，避免大图或视频被半截 chunk 截断。
- **多渠道兼容性修复**: 修复豆包、TS-AI、OpenAI 视频等渠道在模型字段为空时默认模型不生效的问题；豆包/TS-AI 图片接口会强制关闭不兼容的流式路径；TS-AI 图生图现在会正确携带 workflow。
- **视频生成稳定性修复**: 修复豆包视频响应 `content` 为列表时解析崩溃的问题；修复 `.mp4` 签名 URL 下载时 query 参数被截断导致 403 的问题。
- **下载安全增强**: 下载模型返回的视频 URL 时默认不再向第三方 CDN 发送 API Key，仅同域 URL 才附带认证头；图片 URL 下载增加 Content-Type 与魔数校验，避免把普通网页或错误页当图片发送。
- **错误提示与日志优化**: API 成功响应但未返回图片时，会尽量透传模型返回的拒绝/说明文本；网络和 SSE 错误日志补充异常类型与 `repr`，减少空错误信息。
- **输入与配置修复**: 收紧文本 `@数字` 识别规则，减少 `logo@2x`、`@128` 等提示词误判；修复 TOML `admins` 单行/多行数组自动修复状态机不退出的问题。
- **代码与依赖整理**: 清理未使用变量和导入，加载日志改为读取真实插件版本；依赖声明提升为 `httpx>=0.26.0` 以匹配当前 `proxy=` 参数用法。

- **原生 v1.0 运行时适配**: 完善消息兼容包装和配置缓存同步，修复新版插件运行时下部分命令收到 dict 消息对象时访问 `message_info` 报错的问题。

- **视频与渠道管理增强**: 增加视频渠道标记、流式开关、模型更新和 Key 错误上限等管理能力，便于在 WebUI/原生运行时中维护多渠道配置。
